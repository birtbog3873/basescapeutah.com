### Built: md_to_gdoc.py Real Table Support
Discovered the converter script was rendering markdown tables as pipe-delimited text lines instead of actual Google Docs table objects. Fixed by adding a Step 4 to the script that:
1. Deletes the pipe-delimited text at each table's position
2. Inserts a real `insertTable` via the Google Docs API
3. Populates cells in reverse order to preserve index math
4. Bolds header rows with correct post-insertion position calculation

**Key insight:** Google Docs `insertTable` adds a newline before the table, so the table starts at insertIndex + 1. Cell content index formula: `T + 3 + r*(2C+1) + 2c` where T = table startIndex. Derived empirically by creating a test table and reading back the document structure.

Also fixed `gws_batch` to use `subprocess.run` with list args instead of shell=True — the old shell approach would mangle `$` signs and backticks in cell text (e.g., "$1.08M", "$720K").
