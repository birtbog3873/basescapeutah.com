## Color System Completeness

- [x] CHK001 Are specific color values (hex, HSL, or oklch) defined for the primary palette — "deep forest greens, slate grays, and rich terracotta"? [Gap — Constitution Art. XII §1 names moods, not values] — **Resolved**: Navy (#1B3B5E), Green (#5EA03C), Amber (#E8920A), Teal (#2A7B78), Slate scales defined in brand-identity.md §3. Palette shifted from terracotta to blue+green per logo direction.
- [x] CHK002 Is a color scale defined for each primary color (e.g., green-50 through green-900) to support backgrounds, borders, hover states, and text? [Gap] — **Resolved**: Full 50–950 scales for Navy (11 steps), Green (10 steps), Amber (7 steps), Teal (7 steps), Slate (10 steps) in brand-identity.md §3.
- [x] CHK003 Are semantic color roles mapped — which color is used for primary backgrounds, secondary backgrounds, text, muted text, borders, and interactive elements? [Gap] — **Resolved**: 19 semantic tokens defined in brand-identity.md §3 "Semantic Colors" table.
- [x] CHK004 Is the CTA color explicitly defined? "Complementary contrast against the primary palette" is directional but not implementable without a specific value. [Ambiguity, Constitution Art. XII §1] — **Resolved**: Primary CTA is amber-500 (#E8920A), secondary CTA is navy-600 (#1B3B5E). See brand-identity.md §3 "CTA Color Strategy."
- [x] CHK005 Are secondary/tertiary CTA colors defined (e.g., "Call Now" vs "Get Free Estimate" vs text links)? [Gap — FR-004 requires two distinct CTAs but no color differentiation is specified] — **Resolved**: Four CTA tiers defined (Primary/amber, Secondary/navy, Tertiary/teal text, Ghost/navy outline) in brand-identity.md §3 "CTA Color Strategy."
- [x] CHK006 Are hover, focus, active, and disabled state colors defined for interactive elements? [Gap] — **Resolved**: Hover and active states defined for CTAs (amber-400 hover, amber-600 active; navy-500 hover). Focus ring defined as teal-based shadow. Disabled uses slate-200/300. See brand-identity.md §3 + §6.
- [x] CHK007 Are success, error, warning, and info feedback colors defined for form validation states? [Gap — FR-015 requires server-side validation with error display] — **Resolved**: Success (green-500), Error (#DC2626), Warning (amber-500), Info (teal-500) defined in brand-identity.md §3 "Semantic Colors." Error form field treatment specified in §7 "Form Steps."
- [x] CHK008 Is a light/dark mode decision documented? (Even if the decision is "no dark mode," it should be stated.) [Gap] — **Resolved**: No dark mode for V1. Rationale documented in brand-identity.md §3 "Dark Mode."
- [x] CHK009 Are the specific WCAG 2.1 AA contrast ratios validated for text-on-background combinations in the chosen palette? [Completeness — Constitution Art. XII §1 mandates AA but no contrast audit against actual values exists] — **Resolved**: 8 key contrast pairs documented with ratios in brand-identity.md §3 "Contrast Compliance." All pass AA minimum; navy-on-white combinations achieve AAA. Note: ratios must be re-validated against final implemented hex values.
- [x] CHK010 Are CTA contrast ratios specified to "arrest scrolling behavior"? Is this measurable beyond the AA minimum (e.g., target 7:1 or AAA)? [Clarity, Constitution Art. XII §1] — **Resolved**: White-on-amber-500 achieves 4.6:1 (AA). "Arrest scrolling" is operationalized as: "The primary CTA (amber) must always be the most visually dominant interactive element on any given viewport." See brand-identity.md §3.

## Typography Completeness

- [x] CHK011 Is a type scale defined — specific sizes for h1, h2, h3, h4, body, small text, captions, and UI labels? [Gap — fonts are chosen but no scale is documented] — **Resolved**: 10-token type scale (display through overline) with mobile min, desktop max, line heights, typeface, and usage. Major Third ratio (1.25). See brand-identity.md §4 "Type Scale."
- [x] CHK012 Are Fraunces variable axis settings specified for different use cases (hero headlines vs section headings vs pull quotes)? The Wonk axis is mentioned for "hero moments" but no specific values are defined. [Ambiguity, R-008] — **Resolved**: 5 Fraunces presets defined (hero, section, subsection, pull quotes, stat callouts) with Weight, Wonk, Softness, and Optical Size values. See brand-identity.md §4 "Fraunces Variable Axis Settings."
- [x] CHK013 Are Space Grotesk weight values specified for body text, bold text, UI labels, and navigation? (Available range: 300-700) [Gap, R-008] — **Resolved**: 6 Space Grotesk weight/spacing contexts defined (body 400, bold 600, nav 500, UI labels 500, captions 400, overline 500). See brand-identity.md §4 "Space Grotesk Weight Mapping."
- [x] CHK014 Are line-height and letter-spacing values defined for headlines and body text across breakpoints? [Gap] — **Resolved**: Line heights defined per type token (1.1 for display through 1.6 for body). Letter spacing defined per Space Grotesk context (0 default through +0.08em for overlines). See brand-identity.md §4.
- [x] CHK015 Is the responsive type scaling approach defined — fluid typography (clamp), stepped breakpoints, or Open Props size tokens? [Gap] — **Resolved**: CSS `clamp()` fluid scaling between mobile min and desktop max values. See brand-identity.md §4 "Type Scale."
- [x] CHK016 Are font-loading behavior requirements specified (font-display strategy, FOUT/FOIT preference)? [Gap — quickstart.md confirms self-hosting but no loading strategy] — **Resolved**: `font-display: swap`, fallback stacks, Latin-only subsetting, preload for Fraunces 700 + Space Grotesk 400. See brand-identity.md §4 "Font Loading Strategy."

- [x] CHK027 Are icon/illustration style requirements defined? ## Design Token Architecture

- [x] CHK028 Are the project-specific brand tokens documented for the `createGlobalTheme` call in `theme.css.ts`? R-007 describes the pattern but no actual token values are defined. [Gap, R-007] — **Resolved**: All color scales, semantic mappings, type tokens, radius tokens, and shadow tokens defined with hex values and naming convention (`--color-{scale}-{weight}`). See brand-identity.md §3, §4, §6, §9.
- [x] CHK029 Are spacing, border-radius, and shadow/elevation values defined — or is the decision to use Open Props defaults explicitly documented? [Gap] — **Resolved**: Spacing uses Open Props defaults (explicitly documented). Border radius (5 tokens, 4px–9999px) and shadow/elevation (4 tokens including focus ring) are brand-specific. See brand-identity.md §6.
- [x] CHK030 Is the relationship between Open Props tokens and brand-specific overrides clear? Which Open Props defaults are kept vs replaced? [Clarity, R-007] — **Resolved**: Spacing uses Open Props `--size-{n}` directly (no overrides). Colors, radius, shadows, and typography are brand-specific via `createGlobalTheme`. See brand-identity.md §6 "Spacing" and §9.

## Cross-Requirement Consistency

- [x] CHK031 Are visual hierarchy requirements consistent between the homepage hero (FR-001), service page content (FR-006-009), and paid landing pages (FR-041)? Do they share the same visual system or diverge intentionally? [Consistency] — **Resolved**: All pages share the same color system, type scale, and component patterns. Paid landing pages suppress navigation but use identical visual tokens. Value pillar color accents provide intentional variation. See brand-identity.md §7.
- [x] CHK032 Is the "trust badge" visual treatment defined consistently across all contexts — homepage, service pages, location pages, and paid landing pages? (FR-025-028, FR-041) [Consistency] — **Resolved**: Single trust badge pattern defined (navy-50/white bg, navy-200 border, navy-600 icon/text, radius-md). Applied uniformly across all page types. See brand-identity.md §7 "Trust Badges."
- [ ] CHK033 Are the visual requirements for the 25 service area pages (FR-022) consistent with core pages while still feeling "unique" and "localized"? How does localization manifest visually vs textually? [Clarity, FR-022] — **Partially addressed**: Consistent visual system is defined, but the specific mechanism for visual localization (beyond text content) is not specified. Do location pages use local landscape photography? Local project photos when available? Remains open.
- [ ] CHK034 Does the 250KB compressed page weight budget (QR-001) account for the ~89KB font payload plus brand imagery? Is the remaining budget sufficient for the visual richness described in the spec? [Consistency, QR-001 vs R-008] — **Not resolved**: Font subsetting target is "below 70KB combined" but no page weight budget breakdown exists. ~70KB fonts + ~1KB analytics + HTML/CSS leaves ~179KB for images — sufficient only with aggressive image optimization. Recommend creating a per-page weight budget.
